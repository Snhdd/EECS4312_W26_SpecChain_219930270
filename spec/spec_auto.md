# Requirement ID: FR1
- Description: The system shall provide emotionally supportive responses to users experiencing anxiety, distress, or negative thoughts.
- Source Persona: Emotionally Distressed Support User
- Traceability: Derived from review group G1
- Acceptance Criteria: Given a user expresses emotional distress, when the chatbot receives the message, then it must return a supportive response within 2 seconds.

# Requirement ID: FR2
- Description: The system shall provide guided CBT exercises to help users manage negative thought patterns.
- Source Persona: Guided Self-Help User
- Traceability: Derived from review group G2
- Acceptance Criteria: Given a user opens CBT tools, when the feature is selected, then at least one guided CBT exercise must be displayed.

# Requirement ID: FR3
- Description: The system shall provide guided meditation and breathing exercises through the wellness tools interface.
- Source Persona: Guided Self-Help User
- Traceability: Derived from review group G2
- Acceptance Criteria: Given a user selects meditation or breathing exercises, when the feature starts, then the selected exercise must load successfully.

# Requirement ID: FR4
- Description: The system shall maintain relevant conversation context across user sessions.
- Source Persona: Personalized Conversation User
- Traceability: Derived from review group G3
- Acceptance Criteria: Given a returning user continues a prior conversation, when a follow-up message is sent, then the chatbot must reference earlier context.

# Requirement ID: FR5
- Description: The system shall generate contextually relevant responses and reduce repetitive or generic replies.
- Source Persona: Personalized Conversation User
- Traceability: Derived from review group G3
- Acceptance Criteria: Given two different user prompts, when the chatbot responds, then each reply must be specific to the user input and not duplicated.

# Requirement ID: FR6
- Description: The system shall support multilingual interaction for supported non-English languages.
- Source Persona: Inclusive Access User
- Traceability: Derived from review group G4
- Acceptance Criteria: Given a supported language is selected, when the user interacts with the system, then the chatbot and interface labels must appear in that language.

# Requirement ID: FR7
- Description: The system shall support accessible navigation for users relying on assistive technologies.
- Source Persona: Inclusive Access User
- Traceability: Derived from review group G4
- Acceptance Criteria: Given a user relies on screen-reader support, when navigating the main interface, then key functions must remain accessible.

# Requirement ID: FR8
- Description: The system shall clearly display subscription plans, pricing, and premium feature differences.
- Source Persona: Price-Aware Premium Evaluator
- Traceability: Derived from review group G5
- Acceptance Criteria: Given a user opens the premium page, when subscription options are displayed, then pricing and included features must be clearly shown.

# Requirement ID: FR9
- Description: The system shall display free trial and billing information before premium activation.
- Source Persona: Price-Aware Premium Evaluator
- Traceability: Derived from review group G5
- Acceptance Criteria: Given a user starts a free trial, when trial information appears, then billing date and cancellation information must be visible.

# Requirement ID: FR10
- Description: The system shall keep core emotional support features available to non-premium users.
- Source Persona: Price-Aware Premium Evaluator
- Traceability: Derived from review group G5
- Acceptance Criteria: Given a non-premium user accesses the application, when using core support functions, then emotional support chat must remain available.
